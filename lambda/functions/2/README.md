# 採用案
- アップロードした時に、軽量サイズの動画(.mp4、HLS)を作成してS3で保存し続ける。
- 記事が削除されるなどして、必要がなくなったら削除する。

# 却下案
- 動画にアクセス時、Lambda@edgeで軽量サイズの動画(.mp4、HLS)を作成して配信する。
- 長期間アクセスがない場合は、ソースだけ残して加工後ファイル(.mp4、HLS)を削除する。
- 再びアクセスがあった時に、加工後ファイルを作成して配信する。
- 動画にアクセスした時、
    - 加工後の動画があれば、何もしない。
    - 加工後の動画がなければ、
        - 加工中のフラグ(`/tmp/{id}.txt`)が存在するかを確認して、
            - 加工中なら、何もしない。
            - 加工中じゃなければ、
                - MediaConvertにJOBを作成して、
                - 加工中のフラグを立てて、
        - `/videos/waiting.mp4`にリダイレクトする。

## 却下理由
- MediaConvertでファイルを作成するのにコストがかかる。
- クローラーが一斉に動画ファイルにアクセスすると、節約しようとする利点が潰される。
- アクセスがなくても単純にS3でファイルを保存し続けておく方が、コストは安い。
